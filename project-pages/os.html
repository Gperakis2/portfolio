Operating System Project
A project that required a group of 3 to implement some extensions to the kernel of a simple operating system, called tinyos-3, that runs as a process on Linux and uses simulated resources.
Project Details
Simulated computer: The simulated computer that tinyos-3 manages, which offers a low-level API for accessing its resources, such as CPU cores, terminals, clock, etc. The computer offers a low-level API for accessing its resources, implemented by firmware called BIOS.
Tinyos-3 files: The files that contain the code of the current version of tinyos-3 and the ones that need to be extended. The files are categorized into executable files, simulator files, application files, system files, and auxiliary files.
Developing tools: The tools that are needed for developing and testing the code, such as gcc compiler, make utility, tmux program, etc.
Simulated computer diagram: The image shows the schematic diagram of the simulated computer that runs tinyos-3. It consists of cores, interrupts, barrier, inter-core interrupt, programmable interrupt controller (PIC), clock, and universal asynchronous receiver-transmitter (UART).
Priority scheduler: The first part of the assignment was to implement a priority scheduler based on the multilevel feedback queues algorithm. The algorithm uses an array of queues, one for each priority level. Each thread has a priority field, which determines the queue it belongs to. The priority is adjusted according to the thread’s behavior and the first element of the highest priority non-empty queue is selected as the next thread.
Processes: The second part was to convert tinyos-3 processes to multithreaded ones.
The third part was to implement two process communication mechanisms, pipes and sockets.
The final part was to implement a user program that prints information about the system to standard output.
